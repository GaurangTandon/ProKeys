    FAQ

    Why won't this work in Google Docs?
    Google Docs by itself is a very powerful rich text editor. It is built in a very complex manner, which makes it difficult for ProKeys to recognize the caret position and perform relevant functions. Sorry for the trouble!
    I deleted a snippet/made changes by mistake! What do I do?
    Each of your edits/deletions/etc. are saved up carefully and listed in the "Past data versions" on the Snippets tab. You can restore your data from there :-)
    Sync storage breaks. How to fix that?
    This is a known issue. While we try to figure out the causes and fix it, currently a workaround is to:
    1. Make changes to your data in the original sync device and save them.
    2. Wait for a few minutes.
    3. Watch as the child sync devices automatically catch the update and restore their normal operation.
    Is it possible to have a snippet expand without a hotkey?
    Sorry, but given the way ProKeys is built, it is necessary to have a hotkey to execute a snippet.
    Why doesn't ProKeys work in email and password fields?
    Email field doesn't support caret manipulation, so ProKeys can't detect caret movements there, and hence can't function. ProKeys is disabled on password fields for obvious privacy reasons.

    Info

    What are snippets (snips)?

    Snippets are pairs of an abbreviation and a long text. Typing the abbreviation, pressing the hotkey (default: Shift+Space), replaces it with the long text.

    Example: Snippet with snippet name - "brb". The snippet name is the abbreviation. It is case-insensitive, so "brb", "BRB", and "bRb" are all same. Typing "brb" and pressing the hotkey, would replace it with the snippet body, which here is "be right back". Snippets can be created in the popup box.

    Try snippets in editor - you can view the snippets in the popup box
    What are Placeholders?

    Placeholders are fields in the snippet body which are populated with dynamic values provided by you while using the snippet.
    A placeholder has the format `%text%` such that the text can only contain alphabets, digits and the underscore symbol.

    Example: snippet "wishBDay", which you can use to wish your friends Happy Birthday! Since the name of the friend keeps changing, so you can add a placeholder to the snippet body, as "%name%"(without quotes). It would be auto-highlighted whenever you use the snippet and you can type your friend's name into it.

    Try the "letter" sample snippet in editor

    Usage of a placeholder:

    1. Insert placeholders into the snippet body while creating or editing a snippet. Then, use this snippet.
    2. The first placeholder would be auto-highlighted. Type whatever you want to type there.
    3. Press `[Tab]` key to move to the next placeholder.
    4. Repeat steps 2 and 3 till all placeholders are done filling.
    What is Mathomania?

    Often, mathematical calculations require you to leave your current work and use the calculator. But, using Mathomania, you can do calculations within the text editor itself.

    Supported operations are: addition, subtraction, multiplication, division, exponentiation (like: 2^3; of integer constants only), and percentage (using % symbol). You can group expressions using parentheses too.

    Usage:

    1. Press square bracket key twice, the other `]]` pair of braces would auto-complete.
    2. Type the math expression to be evalated inside the braces.
    3. Press `=` (equals) key and the result of evaluation would replace the `[[...=]]`!
    Try math expressions in the editor (like "[[5^3=]]")
    How to Embed one Snippet inside of another?

    Embedding one snippet inside of another allows you to pull the contents of one snippet into another. This is extremely useful when you have a certain text repeated in several snippets. You can declare that certain text as a single separate snippet, and then embed this new snippet inside the other snippets.


    All the placeholders and macros of the embedded snippet work as if they were a part of the parent snippet. You can embed more than one snippet inside a parent snippet, and your embedded snippet can also contain other embedding references.


    To embed a snippet "snp2" inside another snippet "snp1", you would simply type "[[%s(snp2)]]" (without quotes) inside the "snp1" snippet's body.

    What are Date/Time macros?

    Date-Time macros can be used to insert current date and time related data.

    Usage

    They are to be placed inside a snippet body while creating the snippet.

    1. Place "[[%d()]]" (without quotes) inside the snippet body.
    2. Place one or more of the symbols inside the parentheses, from the symbol list.
    3. When you use the snippet, this entire string of `[[%d(...)]]` would be replaced by the current value of the symbols it contains.
    Symbol list
    Symbol to insert Remarks Sample output
    s seconds 40
    m minutes 56
    hh hour (in 24 hour format) 23
    h hour (in 12 hour format) 11
    a am or pm, as per time am
    Do date with suffix 14th
    D date 14
    dddd day of the week Sunday
    ddd day of the week (short name) Sun
    MMMM full month name March
    MMM short month name Mar
    MM numeric month 03
    YYYY year (four digit) 2015
    YY year (two digit) 15
    ZZ complete timezone India Standard Time
    Z abbreviated timezone IST
    z timezone offset wrt GMT +0530
    date complete current date March 05, 2015
    time complete current time (24hour format) 01:29:55
    What is Date/Time arithmetic?

    Date and Time arithmetic lets you move forward or backward in time in the Date/Time macros. It is of two types:

    1. Relative: All the macro symbols inside a parentheses perform calculations relative to the same date, and have a single shared copy of the date. So, calculations on one symbol change that same shared copy, and could affect result of another symbol. Like, on forwarding a date by 10 days (using [[%d(D+10)]]), if the date is caused to move into another month, then that changed month would be used as the current month for any calculation on any month symbol further inside the parentheses. Similar rules apply for changing minutes (causing the hour to change), etc.
    To indicate relative calculation, add a ! at the start, like this: [[%d(! D+5 M-5 ....)]]

    2. Independent (Default): All the macro symbols while performing calculations are independent of one another. So, if forwarding a date by 10 days (using [[D+10]]), causes the date to move into another month, still, that changed month would not be used as the current month for any calculation on any symbol further inside the parentheses. All the symbols would have their own separate copy of the date.

    Usage: After a date/time macro symbol, you have to place a "+" or "-" sign (indicating whether to move forward or backward respectively), followed by a number, which would indicate the amount by which to move forward or backward. No white space or any other character should be used between the symbol, sign, and number. So, [[%d(Do+5 M-10)]] is valid, while [[%d(Do - 5 M- 10)]] is invalid.

    Evaluation of these symbols is from left to right, and the symbol that appears first (at the top) in the symbols list, is evaluated first.

    Example: Consider this date/time macro: [[%d(YYYY:MM:D)]]. If you want to move 10 days forward and 5 months backward, you could write: [[%d(YYYY:MM-5:D+10)]] This was independent calculation. If you want it to be relative, use [[%d(!YYYY:MM-5:D+10)]] (with that !)

    Try date and time arithmetic in editor (use dateArithmetic sample snippet)

    Limitation: The "date" and "time" symbols do not support date and time arithmetic (I might fix this in future).

    How do I use Clipboard Paste Macro?
    Clipboard Paste Macro is used to paste the contents of your clipboard (that data which you have earlier copied or cut) into the snippet body.
    To use it, you only have to place "[[%p]]" (without quotes) in the snippet body, at the location where you want your clipboard data to be pasted.
    How do I use Browser URL macro?
    The Browser URL macro is used to insert the URL (or parts of the URL) of the current webpage into the snippet. Assuming the URL to be "https://www.website.com/path1/path2/path3#hash?a=b&c=d", these are the sample outputs:
    Macro code Remarks Sample output
    (empty) Full URL excluding protocol and www website.com/path1/path2/path3
    (any number) Path till which URL is output website.com/path1/path2 (for n = 2)
    (p) Includes the protocol in the output https://website.com/path1/path2/path3
    (w) Includes the www in the output www.website.com/path1/path2/path3
    (q1) Number of query parameters to be included www.website.com/path1/path2/path3?a=b
    (h) Includes the location hash in the output website.com/path1/path2/path3#hash
    {p} Outputs only the protocol https
    {3} Outputs only the n-th path of the URL path3
    {q2} Outputs only the 2nd query parameter of the URL path3
    {h} Outputs only the location hash hash
    What are Variables?

    Variables are used to insert dynamic values at any time while typing. They are inbuilt and cannot be created by you. Of course, I would love a recommendation from you for a new variable! (email: prokeys.feedback@gmail.com)

    Usage:

    1. Type `[[`, the other pair of braces `]]` would auto-complete.
    2. Type the variable name inside them, and press `=`.
    3. This entire `[[...=]]` would be replaced by the dynamic values of these variables.

    The following Variables (case-insensitive) are supported:

    Variable name Value it holds Sample output
    date current, complete date March 05, 2015
    time current, complete time 01:29:55
    version current browser version Chrome 43.0.2357.130
    Try these variables in editor
    How are Variables different from Snippets?
    Variables are more useful when you need to retrieve system-related dynamic values. Like the version of the browser, which would need to be manually entered into a snippet. A Variable would get it automatically.

    Troubleshoot

    Try reloading the page before anything else. That fixes most issues.
    ProKeys works improperly/does not work on XYZ site
    1. Some sites like Google Docs, Quora, Facebook, etc. insert their custom textboxes which maks it difficult for ProKeys to recognize the caret position and perform relevant functions.
      Unfortunately, there is no workaround for this problem and you can only block the site in Settings for the time being.
    2. Some sites like Evernote insert their own custom <iframe>s in the page which may require a custom, manual check as it difficult to get ProKeys to work with <iframe>s. Please mail me at prokeys.feedback@gmail.com if you have come across any such site and I will try my best to fix ProKeys for that site.
    3. Code/programming editors like Codemirror, Ace, etc. used in variety of sites are not supported since they have their own custom features and prevent ProKeys from working properly.
    4. In case of snippets, make sure your caret(cursor) is placed just after the snippet name. Example: If your snippet name is "abcd", place your caret after "d"
    5. Check that the site is not blocked in Settings.
    6. If nothing works, take a backup of your data and reinstall ProKeys. Restore your data back and try again.
    7. If it still does not work, contact us at prokeys.feedback@gmail.com
    Sync storage does not work
    1. Make sure that you're signed into Chrome. The tab on the upper-right corner of the title bar should not read "Guest".
    2. Data can be synced across one Google account only. Both your Chrome browsers must be signed in with the same Google account.
    3. Click the two sync buttons in the right order - use the first button when you're setting up sync for the first time; the second button when you already have sync set up on another PC.
    4. In case you use multiple email accounts, make sure you're not confusing the ones which you use to sign-in to Chrome.
    5. If it still doesn't work, contact us at prokeys.feedback@gmail.com
